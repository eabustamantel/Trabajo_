ESTRUCTURA GENERAL DEL DOCUMENTO
================================================================================

DOCTYPE y HEAD:
- DOCTYPE html: Define el tipo de documento HTML5
- lang="es": Establece el idioma español para accesibilidad
- Meta charset UTF-8: Codificación de caracteres universal
- Meta viewport: Configuración responsive para dispositivos móviles
- Title: "Taller Valenciano Guayaquil 2025 - Gestión de Inventario"
- Enlaces externos:
  * Google Fonts (Poppins): Tipografía moderna y profesional
  * Font Awesome 6.0.0: Iconos vectoriales para interfaz
  * CSS local: styles.css para estilos personalizados

================================================================================
CARACTERÍSTICAS TÉCNICAS IMPORTANTES
================================================================================

1. ARQUITECTURA SPA (Single Page Application):
   - Una sola página HTML con navegación JavaScript
   - No recarga la página completa al cambiar secciones
   - Mejor experiencia de usuario y rendimiento

2. DISEÑO RESPONSIVE:
   - Meta viewport para adaptación móvil
   - CSS Grid/Flexbox (definido en styles.css)
   - Navegación adaptable a diferentes tamaños de pantalla

3. ACCESIBILIDAD:
   - Atributos alt en imágenes
   - Estructura semántica HTML5 (header, main, section)
   - Navegación por teclado posible

4. SEPARACIÓN DE RESPONSABILIDADES:
   - HTML: Estructura y contenido
   - CSS: Presentación y estilos
   - JavaScript: Comportamiento e interactividad

5. SISTEMA DE ICONOS:
   - Font Awesome para iconografía consistente
   - Iconos representativos para cada sección
   - Mejora la usabilidad visual

================================================================================
FUNCIONES JAVASCRIPT LLAMADAS
================================================================================

- loadPage(page): Cambia entre secciones del dashboard
- showForm(entity): Muestra modal con formulario para crear/editar
- generatePDF(entity): Genera y descarga reporte PDF
- logout(): Cierra sesión y redirige a login
- closeModal(): Oculta el modal de formularios

================================================================================
ESTRUCTURA DE DATOS ESPERADA
================================================================================

Cada tabla espera datos JSON del backend con estructura específica:
- Clientes: {id, nombre, apellido, telefono, email, direccion}
- Inventario: {id, nombre, descripcion, cantidad, precio, categoria}
- Órdenes: {id, descripcion, fecha_creacion, estado, cliente_id, vehiculo_id}
- Empleados: {id, nombre, apellido, puesto, telefono, email}
- Proveedores: {id, nombre, contacto, telefono, email, direccion}
